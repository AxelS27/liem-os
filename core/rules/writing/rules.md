# Writing Rules

These rules govern all copywriting, documentation, and content creation in Liem OS.

## 1. Input-Driven Output
- All content must render input. Do not invent marketing claims, testimonials, or numbers that are not provided in the source brief or corpus.
- Ensure the output strictly answers the requirements of the brief.

## 2. Voice & Tone
- **Active Voice**: Write in an active, direct, and clean voice.
- **No AI Clichés**: Ban typical AI words like "delve", "testament", "unlock", "seamless", "moreover", "furthermore", "realm".
- **Density & Spacing**: Keep paragraphs short (2-3 sentences max). Use bullet points and lists to improve readability.

## 3. Platform Guidelines
- **Documentation**: Clear, logical hierarchy. Use code snippets and file links where helpful.
- **Social Media (X, Threads)**: Start with a strong hook, keep sentences punchy, use appropriate line breaks, and avoid hash-tag stuffing.
- **Newsletters**: Editorial and personal tone. Provide a clear introduction, value-packed body, and single Call-to-Action (CTA).
