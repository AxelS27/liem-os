# Common Rules

These rules apply globally across all domains, tasks, and agents in Liem OS.

## 1. Quality & Design First
- Follow the 5 Iron Laws of Liem OS at all times.
- Prioritize high-performance, responsive interfaces and structured outputs.
- Never compromise on code safety, lint rules, or documentation.

## 2. Workspace Hygiene
- Do not commit or push changes without verification.
- Keep the workspace clean: delete temporary files and scratch scripts when they are no longer needed.
- Use explicit, short conventional commit messages.

## 3. Communication Style
- Keep responses short, concise, and structured.
- Highlight key decisions and open questions clearly.
- Create clickable file links using markdown `[basename](file:///path/to/file)` for any mentioned files.
